# Urban Wildlife India

> A high-performance, dark-mode wildlife photography gallery — built with Hugo, deployed free on GitHub Pages.

**Live site →** [ashukla81992.github.io/urban-wildlife-india](https://ashukla81992.github.io/urban-wildlife-india/)

---

## What this is

A curated gallery of original urban wildlife photography from across India — leopards in Mumbai, owls on wires, macaques in palm fronds. Built to be fast, beautiful, and as theft-resistant as a static site can be.

**Tech stack:** Hugo (extended) · GitHub Pages · GitHub Actions · Pillow (Python)

---

## Features

| Feature | Detail |
|---|---|
| Dark, minimal UI | NHM Wildlife Photographer of the Year inspired |
| Masonry grid | CSS `columns` layout, responsive 1–3 col |
| Per-photo detail pages | Full-bleed stage · EXIF table · species info · prev/next nav |
| PhotoSwipe 5 lightbox | Opens on single-photo zoom button |
| Category filters | JS-powered, works with multi-category photos |
| Image protection | CSS `background-image` (no right-click Save As) · transparent shield overlay · keyboard shortcut blocking |
| Watermark baked in | Python script embeds handle into pixel data before upload |
| EXIF stripped | GPS & metadata removed on export |
| WebP output | Hugo image processing + Pillow both output WebP |
| Auto-deploy | GitHub Actions builds & deploys on every push to `main` |

## Quick start

### Prerequisites

| Tool | Version |
|---|---|

## Adding new photos

### Step 1 — Drop originals into `raw_photos/`

```
raw_photos/
    kingfisher-dive.jpg
    spotted-deer-roadside.png
    ...
```

This folder is `.gitignored` — your high-res originals never leave your disk.

### Step 2 — Run the watermark script

```bash
# Activate your shared Python venv
source ~/Personal\ Repos/ContentVideoGenrator/activate_shared_venv.sh

# Process all images in raw_photos/
python3 scripts/watermark.py
```

For each image this creates a **page bundle** under `content/gallery/<slug>/`:

```
content/gallery/kingfisher-dive/
    cover.webp      ← watermarked, EXIF-stripped, 85% quality WebP
    index.md        ← stub front matter — fill this in
```

### Step 3 — Fill in the metadata

Open the generated `index.md` and complete the front matter:

```yaml
---
title: "Kingfisher Mid-Dive"
date: 2025-09-14
location: "Powai Lake, Mumbai"
species: "Common Kingfisher (Alcedo atthis)"
camera: "Sony A7 IV"
lens: "200–600mm f/5.6–6.3 G"
settings: "1/2000s · f/6.3 · ISO 1600"
caption: "A male kingfisher hits the water at 40 km/h."
categories: ["birds", "waterbirds"]
tags: ["kingfisher", "alcedo atthis", "urban lake", "dive"]
description: "SEO meta description here (150–160 chars)."
draft: false
---

Story / observation notes go here as Markdown.
```

### Step 4 — Preview, then push

```bash
hugo server --buildDrafts   # verify locally

git add content/gallery/kingfisher-dive/
git commit -m "Add: Kingfisher mid-dive, Powai Lake"
git push
```

GitHub Actions builds and deploys automatically — live in ~2 minutes.

## Deploying to GitHub Pages

### First time

1. Push to GitHub:
```bash
git remote add origin https://github.com/Ashukla81992/urban-wildlife-india.git
git push -u origin main
```

2. In your GitHub repo go to **Settings → Pages → Build and deployment → Source** and select **GitHub Actions**.

The workflow handles everything else — no manual build step needed.

### Every update after that

```bash
git add .
git commit -m "Add: <description>"
git push
```

## Image protection layers

No browser-side protection is absolute, but these layers stop the vast majority of casual copying:

| Layer | How it works |
|---|---|
| **CSS `background-image`** | Right-clicking a gallery tile shows no "Save Image As" — the image URL is in CSS, not in an `<img>` tag |
| **Transparent shield div** | A `position:absolute` div sits above the background, so any right-click hits the div, not the image |
| **`shield.js`** | Blocks context menu on gallery surfaces; blocks `Ctrl/Cmd+S`, `F12`, DevTools keyboard shortcuts |
| **`pointer-events: none`** on `<img>` | Prevents drag-to-desktop of any `<img>` elements (e.g. inside PhotoSwipe) |
| **Watermark in pixel data** | `watermark.py` burns your handle into the image before it ever reaches the server — survives any download |
| **EXIF stripped** | No GPS location or camera metadata in published files (`exif=b""` in Pillow) |
| **Medium-res lightbox** | PhotoSwipe opens a 1200px version, not the full master |

## Configuration

Edit `hugo.toml` to customise:

```toml
title  = "Urban Wildlife India"          # site title

[params]
  description = "..."                    # meta description
  author      = "@urbanwildlifeindia"    # shown in footer
  watermark   = "@urbanwildlifeindia"    # burned into images
```

Edit `scripts/watermark.py` to change:

```python
WATERMARK_TEXT = "@urbanwildlifeindia"   # your handle
WEBP_QUALITY   = 85                      # 1–100
FONT_RATIO     = 1 / 35                  # watermark size relative to image width
```
